# Enable Because Future - Project Context

## Project Overview
This is a full-stack fashion/wardrobe management application with three main components:
1. **Backend API** (Flask/Python)
2. **Frontend Web App** (React)
3. **Chrome Extension** (JavaScript)

The application appears to focus on wardrobe management, garment analysis, and background removal for clothing items.

## Technology Stack

### Backend (`/backend/`)
- **Framework**: Flask (Python)
- **Database**: MySQL
- **Key Dependencies**: 
  - `flask-cors` for CORS handling
  - `mysql-connector-python` for database connectivity
  - `requests` for API calls
  - `opencv-python` for image processing
  - `beautifulsoup4` for web scraping
- **Port**: 5000 (default Flask port)

### Frontend (`/frontend/`)
- **Framework**: React 19.1.1
- **Build Tool**: Create React App
- **Port**: 3000 (development server)
- **Proxy**: Configured to proxy API calls to `http://localhost:5000`

### Chrome Extension (`/chrome-extension/`)
- **Manifest Version**: 3
- **Name**: "Enable Chatbot"
- **Permissions**: activeTab, scripting, storage, all URLs
- **Components**: popup, background service worker, content scripts

## Key Features

### Backend API Endpoints
- `/api/remove-bg` - Background removal service using external API
- Wardrobe management functionality
- MySQL database integration
- Image processing capabilities

### Chrome Extension Features
- Popup chatbot interface
- URL reading capability
- Image upload functionality
- Active tab interaction

### Frontend Features
- React-based user interface
- Wardrobe management
- Image handling and display

## External Services
- **Background Removal API**: `https://api.becausefuture.tech/bg-service/api/remove`
  - Credentials: `becausefuture` / `becausefuture!2025`
  - Used for removing backgrounds from clothing images

## Database Schema
- MySQL database named `hello_db`
- Schema defined in `mysql_setup.sql`
- Environment variables for connection:
  - `MYSQL_HOST` (default: localhost)
  - `MYSQL_USER` (default: root)  
  - `MYSQL_PASSWORD` (default: root)
  - `MYSQL_DATABASE` (default: hello_db)

## Development Workflow
1. **Backend**: Run `python app.py` in `/backend/` directory
2. **Frontend**: Run `npm start` in `/frontend/` directory  
3. **Chrome Extension**: Load unpacked extension from `/chrome-extension/` directory
4. **Database**: Set up MySQL using `mysql_setup.sql`

## File Upload Handling
- Supported formats: PNG, JPG, JPEG, WEBP
- Wardrobe images stored in `/frontend/public/images/wardrobe/`
- Background removal processing for uploaded images

## Common Patterns & Conventions
- Flask CORS enabled for cross-origin requests
- React proxy configuration for API communication
- Chrome extension uses Manifest V3 format
- Image processing with OpenCV and NumPy
- Web scraping with BeautifulSoup for URL-based garment extraction
